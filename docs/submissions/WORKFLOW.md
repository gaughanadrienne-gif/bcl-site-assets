# Submissions workflow (owner-run, no backend)

End-to-end loop from a Squarespace form submission to a live listing to
expiry, and back.

## 1. Form submission arrives at hello@

The `/post-a-job` or `/list-a-rental` Squarespace native form (specs in
`post-a-job-form.md` / `list-a-rental-form.md`) emails the submission to
hello@. Nothing is stored or published automatically -- Squarespace forms
have no API hook into this repo.

## 2. Owner validates

Before doing anything else, check:

- **Is this a real employer/landlord?** (Not a scraper-bait or obviously fake
  listing.)
- **Not a scam** (rentals: no wire-transfer/gift-card/crypto/"sight-unseen"
  red flags; jobs: not MLM/pay-to-start/commission-only).
- **Fair-housing-clean** (rentals: no protected-class exclusions or
  preferences in the description).
- **Authorization checkbox was actually checked** on the form.

If it fails any of these, reply to the submitter (or just don't act) --
nothing goes further.

## 3a. Fast path: edit the manual partial directly

For a submission the owner is confident about, hand-edit
`partials/manual-jobs.json` or `partials/manual-rentals.json` directly:

```json
{
  "entries": [
    {
      "title": "Line Cook",
      "employer": "Boulder Creek Diner",
      "city": "Boulder Creek",
      "url": "https://example.com/apply",
      "salary_text": "$20/hr",
      "submitted_at": "2026-07-19",
      "renewed_at": null
    }
  ]
}
```

(Use the raw-field mapping tables in the form-spec docs -- the keys are the
parser-output keys `normalize_job`/`normalize_rental` already expect.) For a
rental, only set `"postal_code": "95006"` after independently confirming the
address is in 95006 (see the 95006 gate note in `list-a-rental-form.md`);
otherwise leave it unset and set `"city": "Boulder Creek"` so it queues
for a second look instead of silently rejecting.

## 3b. Review-board path: queue + approve + promote

For a batch of submissions, or ones the owner wants a second look at before
committing:

1. Add the submission as an item to `review/<tool>-pending.json` (same shape
   the refresh pipelines already write: raw fields + an `id`).
2. Open `review/review-board.html` (regenerated by `refresh_jobs.py` /
   `refresh_rentals.py`), tap **Approve** on the items to promote, tap
   **Copy approved**, and paste the copied ids (one per line) into
   `review/<tool>-approved.txt`.
3. Run the promotion script:

   ```
   python scripts/promote_submissions.py jobs
   python scripts/promote_submissions.py rentals
   ```

   This reads `review/<tool>-pending.json` + `review/<tool>-approved.txt`,
   stamps each approved item with `submitted_at=<today>` /
   `renewed_at=null`, and appends it to `partials/manual-<tool>.json`
   (deduped by id -- re-running with the same approval file never
   duplicates an entry).

## 4. Next scheduled refresh merges it -- gates still apply

`refresh_jobs.py` / `refresh_rentals.py` load the non-expired manual entries
(`load_manual_entries`, 30-day TTL for jobs / 14-day TTL for rentals) and run
them through the **exact same** `normalize_job`/`include_job` (jobs) or
`normalize_rental`/`include_rental`/`safety_status` (rentals) gates as any
scraped source. A human approving a submission does not bypass geography,
exclusion, 95006, or scam/fair-housing checks -- an owner mistake (wrong
city, missed MLM keyword, unconfirmed ZIP) still queues instead of
publishing.

## 5. It appears on the board

Once published, the entry shows up in `data/jobs.json` / `data/rentals.json`
exactly like a scraped listing, tagged `"source": "Community submission"`.

## 6. Expiry drops it unless renewed

30 days (jobs) / 14 days (rentals) after `submitted_at` (or `renewed_at`),
the entry stops appearing in the published/queued output. It is **not**
deleted from `partials/manual-<tool>.json` -- to renew, the submitter
contacts hello@ again and the owner bumps `renewed_at` to today's date on
the existing entry (fast path, no re-entry of the other fields).

## Deferred (not built in this pass)

- A secure contact-relay (currently: reply-to-hello@ manually).
- Automated renewal-reminder emails at day 10/13.
- "Verified owner/manager" badge issuance on published listings.
- The actual Squarespace form build (owner-only UI, from the specs above)
  and wiring `/post-a-job` + `/list-a-rental` navigation.
